# 🖥️ Interactive Web UI Mockups: Student Activity & Registrar Assistant System

ยินดีต้อนรับสู่ระบบจำลองหน้าจอผู้ใช้งาน (Interactive Web UI Mockups) สำหรับ **ระบบผู้ช่วยกิจกรรมนักศึกษาและงานทะเบียนกิจกรรม คณะวิทยาศาสตร์ มหาวิทยาลัยอุบลราชธานี**

โฟลเดอร์นี้รวบรวมไฟล์จำลองหน้าเว็บอินเตอร์เฟสแบบ Static HTML/CSS/JS ทั้งหมดของระบบ เพื่อให้นักศึกษา อาจารย์ที่ปรึกษา และเจ้าหน้าที่งานทะเบียน สามารถทดสอบรูปแบบการจัดวาง (Layout), ทางนำทาง (Navigation) และฟังก์ชันการทำงานเบื้องต้นได้แบบเสมือนจริง

---

## 🚀 วิธีเปิดใช้งาน Mockup
หน้าเริ่มต้นสำหรับเข้าชม Mockup ทุกหน้าจอคือไฟล์ **`index.html`** คุณสามารถเข้าชมได้ด้วยวิธีต่อไปนี้:

### วิธีที่ 1: เปิดไฟล์ตรง ๆ ผ่านเว็บเบราว์เซอร์
* ดับเบิลคลิกเปิดไฟล์ **`index.html`** บนเบราว์เซอร์ของคุณได้ทันที

### วิธีที่ 2: รัน Local Server ด้วยคำสั่ง (แนะนำ)
เปิด Terminal ในโฟลเดอร์นี้และใช้คำสั่งรันเซิร์ฟเวอร์จำลอง:
* **Node.js (npx):**
  ```bash
  npx serve .
  ```
* **Python:**
  ```bash
  python -m http.server 8000
  ```
  จากนั้นเปิดเบราว์เซอร์เข้าสู่: `http://localhost:8000/`

## 📂 โครงสร้างหน้าจอ Mockup (ตามบทบาทผู้ใช้งาน)

### 💻 1. ฝั่งเจ้าหน้าที่ทะเบียน (Registrar Portal)
* **หน้าหลักงานทะเบียน:** `registrar-home.html` - แดชบอร์ดสรุปกิจกรรมและคิวตรวจสอบ
* **ส่งออกข้อมูล:** `registrar-export.html` - ค้นหา ตรวจสอบรายชื่อ และส่งออกไฟล์ Excel ไปยังระบบ UBU SAC
* **แก้ไขหน่วยกิตรายบุคคล:** `registrar-adjust.html` - บันทึกและปรับปรุงชั่วโมงกิจกรรมรายบุคคลพร้อมเหตุผลการปรับปรุง
* **สร้าง/แก้ไขกิจกรรม:** `registrar-event-form.html` - ตั้งค่าพิกัดเช็คอินด้วย Geofencing (GPS) และกำหนดค่าชั่วโมงกิจกรรม
* **ตรวจหลักฐานกิจกรรมเปิดกว้าง:** `registrar-verify-review.html` - ตรวจข้อมูลที่นักศึกษากรอกเอง (ไม่ใช้ OCR) และรายชื่อกลุ่มจิตอาสาที่ OCR สกัดจากตาราง แก้ไขได้ก่อนอนุมัติ พร้อมคุมเพดาน 9 ชม./วัน
* **ตรวจสอบ/ล็อกโควต้า:** `registrar-quotas.html` - ล็อกประวัติการทำกิจกรรมนอกสถานที่เมื่อครบ 12 หน่วยกิตต่อปีการศึกษา
* **จัดการรอบปีการศึกษา:** `registrar-cycles.html` - จัดเก็บข้อมูลเดิมลงฐานข้อมูลประวัติและเริ่มรอบปีการศึกษาใหม่ด้วยตนเอง
* **ตั้งค่าการแจ้งเตือน Slack:** `registrar-slack.html` - เชื่อมโยงบัญชี Slack ของเจ้าหน้าที่เพื่อรับการแจ้งเตือนจากบอท

### 🎓 2. ฝั่งนักศึกษา (Student Portal)
* **หน้าหลักนักศึกษา:** `student-home.html` - แสดงประวัติการเข้าร่วม ชั่วโมงสะสม และประวัตินำเข้าระบบ UBU SAC
* **เช็คอินเข้าร่วมกิจกรรม:** `student-checkin.html` - เช็คอินด้วย GPS (คำนวณระยะห่างผ่าน PostGIS spatial query — `ST_DWithin`) หรือส่งภาพเซลฟี่/กรอกรหัสยืนยัน
* **ส่งหลักฐานกิจกรรมเปิดกว้าง:** `student-proof-upload.html` - เลือกช่องทาง (สมุดบันทึก/กิจกรรมเดี่ยว/กลุ่มจิตอาสา) กรอกข้อมูลกิจกรรมเองแล้วแนบรูปให้เจ้าหน้าที่ตรวจสอบ (ไม่ใช้ OCR ยกเว้นตารางรายชื่อกลุ่ม)
* **แชทบอท SciU-Buddy:** `student-chatbot.html` - ระบบช่วยเหลือและตอบคำถามทั่วไป พร้อมระบบสำรองติดต่อเจ้าหน้าที่/Slack Alert

### ⚙️ 3. ฝั่งผู้ดูแลระบบ (Admin Portal)
* **หน้าหลักผู้ดูแลระบบ:** `admin-home.html` - แดชบอร์ดสถานะของระบบ
* **จัดการสิทธิ์และผู้ใช้งาน:** `admin-users.html` - สร้างผู้ใช้งาน และกำหนดสิทธิ์การใช้งาน
* **ตั้งค่าระบบทั่วไป:** `admin-settings.html` - จัดเก็บโทเค็นและตั้งค่า SMTP อีเมลสำหรับผู้ดูแลระบบ
* **ตรวจสอบการทำรายการ (Audit Log):** `admin-audit.html` - ติดตามกิจกรรมและความเคลื่อนไหวของผู้ใช้งาน
* **จัดการ FAQ แชทบอท:** `admin-faq.html` - เพิ่ม/แก้ไข/ปิดใช้งานคำถามที่พบบ่อย ที่ SciU-Buddy ค้นด้วย semantic search (bge-m3) — LLM เป็น local model (typhoon2.5-qwen3-4b) ค่าคงที่ ไม่มี Provider ให้สลับ
* **จัดการช่องทางติดต่อ:** `admin-contacts.html` - จัดการรายชื่ออาจารย์/เจ้าหน้าที่ผู้ประสานงาน ที่บอทค้นตรงด้วย SQL Direct Search

### 👥 4. ฝั่งสโมสรนักศึกษาและผู้ประสานงาน (Lead Org / Field Staff)
* **ผู้ประสานงานกิจกรรม (Lead Org):** `lead-org-home.html`
* **เจ้าหน้าที่ภาคสนาม (Field Staff):** `field-staff-home.html`
